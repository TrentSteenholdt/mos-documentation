    Glossary

    Roles and Access

    • Administrator/Owner: Highest association admin role. Can manage members, groups, and system settings.
    • Reviewer: Elevated user who can assess or review content for others (exact permissions vary by page).
    • Member/User: Standard account with access to member-facing features.
    • Admin Level: Numeric access level stored in session (e.g., 0 = user, 1+ = admin; certain features require 3+). Determines access to admin pages and controls.

    Organizations and Groups

    • Group/Association: Your primary organisation within MOS. Many features (News, Reports, Assessments) are scoped to an association.
    • Subgroup: A subdivision within an association used for targeting (e.g., region, squad, team). Membership in subgroups controls visibility of targeted content like News.

    Assessments and Performance

    • Assessment: A review or evaluation of match official performance created by reviewers based on match observations.
    • Diary Entry: A personal record of a match created by an official. Contains observations and self-reflection.
    • Plan: A development or training plan. Includes goals, actions, and timeframes.
    • Assessments: Evaluations such as exams or reviews completed by members within the system.

    Content and Communication

    • News: Articles posted by admins for the association and its subgroups. Sorted by last updated; paged with a Show more button.
    • Email-to-News: Sending an email to a designated mailbox (shown on the News page for admins) to create an article. Subject is the title, body is the content, and attachments are added.

    Files and Media

    • Attachments: Files associated with a record (e.g., News, Media). May be stored in Azure Blob Storage or local uploads, with links routed through file.aspx.
    • Media Library: Read-only browsing of media (e.g., videos, images) linked to reviews/assessments.
    • Folders (File Manager): Admin-facing area for creating, uploading, moving, and sharing files/folders.
    • Share Link: A predefined or generated URL that provides access to a file or resource, sometimes with restricted scope/time.

    System Features

    • Sanitization: Unsafe HTML is removed before rendering to protect users. Links are preserved where possible.
    • Time-ago Display: Relative time formatting for dates (e.g., "Last updated 2 hours ago").
    • Data Storage: All data is stored securely in Microsoft Azure in the Australia Southeast region, ensuring data residency compliance.
