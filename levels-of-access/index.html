    Levels of Access

    Match Official System comes with different levels of access that are controllable by group administrators. This page describes each access level and the permissions granted.

    Access Level

    Administrator-controlled setting

    Overview

    Access in the system is role-based. Each role grants specific permissions to features, data, and administrative functions. Access levels are assigned by group administrators when adding members to a group.

    Access Levels and Permissions

    The system includes four access levels:

    User

    Description: Standard member access with personal workspace

    Permissions:

    • Create diary and plan entries
    • Search users in their group(s)
    • Sit an exam
    • Access folders in their group(s), but restricted to their personal user folder
    • Upload files to personal user folder*

    Viewer

    Description: Read and review access with reporting capabilities

    Permissions:

    • Create diary and plan entries
    • Search users in their group(s)
    • Create and assign match day, media, season, and general review entries on a user
    • Run reporting on the group
    • View review history of a user account
    • Sit an exam
    • Access folders in their group(s), but restricted to subgroup(s) if not also a User at that level
    • Upload files to personal user folder*

    Reviewer

    Description: Assessment review and group folder access

    Permissions:

    • Create diary and plan entries
    • Search users in their group(s)
    • Create and assign match day, media, season, and general review entries on a user
    • Run reporting on the group
    • View review history of a user account
    • Review diary and plan entries that are awaiting review
    • Sit an exam
    • Access folders in their group(s), but restricted to subgroup(s) if not also a User/Viewer at that level
    • Upload files to all subgroup folders they are a reviewer too

    Owner

    Description: Full administrative control

    Permissions:

    • Create diary and plan entries
    • Search users in their group(s)
    • Create and assign match day, media, season, and general review entries on a user
    • Run reporting on the group
    • View review history of a user account
    • Review diary and plan entries that are awaiting review
    • Create, edit, and delete users
    • Modify group settings - disable/enable notifications, enable user registration, etc.
    • Modify assessment templates - customise diary and match day entry templates
    • Manage exam questions and answers
    • Sit an exam
    • Access folders in their group(s) and any subsequent subgroup(s)
    • Upload files to all group folders

    Understanding Folder Access

    How folder access is determined

    Folder permissions are based on your access level and which subgroups you belong to. Some groups may disable personal user folders; if disabled, personal uploads are not available.

    Role-based folder permissions

    User

    • Access only to personal folder: User/{username} (if enabled).
    • Upload and delete files only inside their personal folder.
    • Cannot create, rename, move, or delete folders outside their personal folder.

    Viewer

    • Inherits User permissions.
    • Can view subgroup folders they’re assigned to (read-only).
    • Personal uploads allowed only when user folders are enabled.
    • Cannot create, rename, move, or delete folders.

    Reviewer

    • Inherits Viewer permissions.
    • Can view and upload to group and subgroup folders they have access to.
    • Cannot create or delete folders (folder creation is reserved for Owners/admins).

    Owner

    • Full folder control across the group and its subgroups.
    • Can create, rename, move, and delete folders and files (system-managed folders such as core subgroup and user folder containers may be protected from deletion).
    • Can generate share links for files.

    Typical folder layout

    Home/
├── user/                   (personal user folders, if enabled)
│   ├── john.smith/
│   ├── jane.doe/
│   └── ...
├── subgroups/              (if enabled)
│   ├── Northern Division/
│   ├── Southern Division/
│   └── ...
├── Documents/
├── Photos/
└── Resources/

    Folder Settings

    Group administrators can enable or disable:

    • User Folders: Allow members to have personal storage
    • Subgroup Folders: Organize members into subgroup folders

    When disabled, members cannot access or see those folder areas.

    Permission Hierarchy

    Access levels build upon each other:

    User (basic) → Viewer (read/review) → Reviewer (all reviews) → Owner (administrative)

    Each higher level includes all permissions of lower levels, plus additional capabilities.

    Common Permission Scenarios

    Can I upload files?

    • User: Only to your personal folder
    • Viewer: Only to your personal folder
    • Reviewer: To all group and subgroup folders
    • Owner: To all group and subgroup folders

    Can I delete files?

    • User: Only files you created in your personal folder
    • Viewer: No
    • Reviewer: No
    • Owner: Yes, any file in any folder

    Can I review assessments?

    • User: No
    • Viewer: No
    • Reviewer: Yes, for assessments assigned to you
    • Owner: Yes, all assessments

    Can I create exams?

    • User: No (can only sit exams)
    • Viewer: No
    • Reviewer: No
    • Owner: Yes, and manage exam questions

    Can I manage users?

    • User: No
    • Viewer: No
    • Reviewer: No
    • Owner: Yes, full user management

    Requesting Higher Access

    If you need higher access level permissions:

    1. Contact your group administrator
    2. Explain what additional features you need
    3. They can upgrade your access level in User Management
