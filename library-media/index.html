    Media: View Files from Your Features

    Access Level

    All users

    Overview

    The Media page is a read-only browser for viewing files and media attached to assessments, news articles, and other system features. This is not a file upload area - it shows media that's been attached to items throughout the system such as:

    • Assessment evidence - Photos, videos, or documents attached to your assessments
    • News attachments - Files included with news articles
    • Profile pictures - Member profile images
    • Assessment templates - Documents linked to assessment reviews
    • Other media associated with features

    How to Access: Click Library then Media

    What You Can Do:

    • Browse media and files in a hierarchical folder structure
    • View file details (name, size, type, date)
    • Download individual files
    • Generate secure share links for files
    • Navigate to related assessments
    • (Administrators) Clean up unused profile images

    What You Cannot Do:

    • ‚ùå Upload new files (attach files directly to the feature instead - assessments, news, etc.)
    • ‚ùå Create new folders
    • ‚ùå Delete or rename files
    • ‚ùå Move files between folders

    Understanding the Structure

    Media appears in the Media browser when:

    1. Files are attached to assessments (evidence, photos, documents)
    2. Files are attached to news articles
    3. Profile pictures are uploaded by users
    4. Content is shared as part of assessment feedback or reviews

    These files are organized by feature type and association, not by direct uploads to a "media folder."

    How to Browse Media

    Navigation

    Breadcrumb Bar:

    • Shows your current location (e.g., Home > Assessments > 2024)
    • Click any segment to jump to that level
    • Use Up (‚¨ÜÔ∏è) to go up one level
    • Click Home (üè†) to return to the top

    Viewing Folders

    1. Double-click a folder to open it
    2. Or click the folder name and press Enter
    3. Contents load showing files and subfolders
    4. Breadcrumb updates to show your location

    Viewing Files

    File Information:

    • Name: Full filename with extension (e.g., match-report-2024.pdf)
    • Size: File size in KB, MB, or GB
    • Date Modified: When the file was last updated
    • Type Icon: Visual indicator of file type (üìÑ document, üñºÔ∏è image, üé• video, etc.)

    File Types You Might See:

    • Documents: PDF, DOCX, DOC, TXT, RTF
    • Spreadsheets: XLSX, XLS, CSV
    • Presentations: PPTX, PPT
    • Images: JPG, PNG, GIF, BMP, SVG
    • Videos: MP4, AVI, MOV, WMV
    • Archives: ZIP, RAR
    • Others: Various file types your group has attached

    Sorting:

    • Click column headers to sort (Name, Size, Date Modified)
    • Click again to reverse order (ascending ‚Üë / descending ‚Üì)
    • Folders typically appear before files

    Empty Folders

    If a folder shows "No items in this folder":

    • The folder exists but has no files yet
    • No media has been attached to that feature
    • Or you may not have permission to view all content

    Downloading Files

    Single File Download

    1. Click the file name
    2. Your browser's download dialog appears
    3. Choose save location or accept default (Downloads folder)
    4. Click Save or Download
    5. Result: File downloads to your computer

    Browser Download Behavior

    • Chrome: Auto-downloads to Downloads folder
    • Firefox: Asks where to save or uses Downloads folder
    • Edge: Auto-downloads or asks, depending on settings
    • Safari: Opens download dialog or saves automatically

    For Large Files (over 10 MB):

    • Allow extra time for download to complete
    • Use Wi-Fi rather than mobile data if possible
    • Do not close your browser until download finishes

    After Downloading:

    1. Check your browser's Downloads folder
    2. Open the file with appropriate software
    3. The file is a copy; original remains in the Media browser

    Troubleshooting Downloads

    Download button doesn't work:

    1. Refresh the page and try again
    2. Try a different browser
    3. Check if browser pop-up blocker is enabled
    4. Disable browser extensions temporarily

    Permission denied error:

    1. Verify you have access to the item (news, assessment, etc.)
    2. Check your group membership
    3. Log out and back in
    4. Contact your administrator if access should be granted

    File missing:

    1. File may have been removed
    2. Sharing link may have expired
    3. Contact your administrator

    Share Links

    Create a Share Link

    Share links allow you to provide access to specific files without requiring recipients to sign in.

    How to Create:

    1. Select the file you want to share (click once to highlight)
    2. Click the Share button in the toolbar
    3. Share Link Modal opens

    Choose Link Expiration:

    • 1 Day: Link expires after 24 hours
    • 7 Days: Link expires after one week (recommended)
    • 30 Days: Link expires after one month
    1. Click Generate Link or Create Share Link
    2. A unique URL is generated
    3. Click Copy to Clipboard or manually copy the URL

    Sharing the Link

    • Paste into email, chat, or message
    • Recipients click the link to download the file
    • No MOS account required (during validity period)
    • Link expires automatically after the chosen period

    What Recipients See:

    • Download page with file name and size
    • Download button to get the file
    • Expiration date
    • No access to other files or your library

    Security Best Practices:

    • ‚úÖ Use shortest expiration that meets your needs
    • ‚úÖ Only share links with intended recipients
    • ‚úÖ Avoid posting links publicly
    • ‚úÖ Generate new link rather than re-sharing old ones
    • ‚ùå Don't share links in public forums or social media

    Revoking Links

    If you need to revoke a link before expiration:

    • Contact your administrator
    • They can manually revoke specific links
    • Link immediately becomes invalid

    Assessment-Related Media

    View Assessment Evidence

    Some folders in Media contain files linked to assessments:

    How to Find:

    1. Navigate to the assessments folder in Media
    2. Look for a View Assessment link or button
    3. Click to open the full assessment with all details and evidence

    What You'll See:

    • Original assessment details
    • Reviewer feedback (if reviewed)
    • All attached media (photos, videos, documents)
    • Rating scales and comments

    Downloading Evidence:

    1. Open the assessment
    2. Look for attachment links or download buttons
    3. Click to download the file
    4. Results appear in your Downloads folder

    Profile Images Cleanup

    Admin only: This feature is restricted to Owners and Administrators.

    Unused Profile Images

    Over time, your MOS group may accumulate profile images for users who no longer exist or have changed their profile picture multiple times, leaving orphaned image files.

    What This Tool Does:

    • Scans the media library for profile image files
    • Identifies images not currently assigned to any user profile
    • Allows bulk deletion of these orphaned images

    Access the Tool:

    1. Navigate to Library ‚Üí Media
    2. (Admins only) Admin Tools panel appears
    3. Click Profile Images Cleanup or Clean Unused Profile Images

    Review Orphaned Images:

    1. A list of unused profile images appears:

      • Image thumbnail
      • File name
      • File size
      • Upload date
      • "Not in use" indicator

    2. Select images to delete (checkboxes)

    3. Or Select All to delete all unused images

    Delete Images:

    1. Click Delete Selected or Remove Unused Images
    2. Confirmation prompt appears: "Are you sure? This cannot be undone."
    3. Confirm to permanently delete

    Results:

    • Selected images are removed from the media library
    • Storage space is freed
    • No impact on current user profiles (only orphaned files are removed)
    • Success message shows how many images were deleted

    When to Use:

    • Storage space is running low
    • Performing regular media library maintenance
    • After bulk user deletions or profile migrations
    • To clean up old test accounts

    Caution:

    • ‚ö†Ô∏è Deletion is permanent - images cannot be recovered
    • ‚ö†Ô∏è Ensure images are truly orphaned before deleting
    • ‚ö†Ô∏è If in doubt, do not delete - orphaned images don't cause harm
    • ‚ö†Ô∏è Consider downloading a backup before bulk deleting

    Tips:

    • ‚úÖ Run cleanup quarterly or annually as part of maintenance
    • ‚úÖ Review list carefully before bulk delete
    • ‚úÖ Only delete if storage is a concern; orphaned images are harmless
    • ‚ùå Don't delete images that might be in use elsewhere (linked in news, assessments, etc.)

    Troubleshooting

    Can't See Expected Files

    Problem: Files or folders are missing from the media library

    Solutions:

    1. Check Your Location:

      • Verify you're in the correct folder (check breadcrumb)
      • Navigate to parent folder and look for subfolder

    2. Refresh the View:

      • Click Refresh button to reload folder contents
      • Files may have been added since you opened the page

    3. Permissions:

      • You may not have permission to view certain folders or files
      • Contact your administrator if you think you should have access

    4. Subgroup Visibility:

      • Some files may be restricted to specific subgroups
      • Check with administrator about subgroup assignments

    5. Files May Be in Folders (not Media):

      • Media is read-only view; some content may only be in Folders page
      • Try Library ‚Üí Folders instead

    Download Doesn't Start

    Problem: Clicking download doesn't download the file

    Solutions:

    1. Browser Blocked Download:

      • Check browser's download/popup blocker
      • Allow downloads from MOS domain
      • Try again

    2. File Too Large:

      • Large files may time out
      • Try a wired connection or better Wi-Fi
      • Contact admin if file is too large for normal download

    3. Browser Compatibility:

      • Try a different browser (Chrome, Firefox, Edge)
      • Update your current browser to latest version

    4. Network Issues:

      • Check your internet connection
      • Try again later if network is slow

    Share Link Doesn't Work

    Problem: Generated share link shows error or doesn't load

    Solutions:

    1. Link Expired:

      • Check expiration date
      • Generate a new link if needed

    2. Link Copied Incorrectly:

      • Ensure entire URL was copied (check for truncation)
      • Re-copy from share modal

    3. Permissions Changed:

      • File may have been deleted or moved
      • Permissions may have been revoked
      • Generate new link

    4. Recipient Issues:

      • Recipient may have network/browser issues
      • Ask them to try different browser or device

    Tips

    For All Users:

    • ‚úÖ Use breadcrumb navigation to jump between folders quickly
    • ‚úÖ Sort by Date Modified to find recent uploads
    • ‚úÖ Generate short-lived share links (1-7 days) for better security
    • ‚úÖ Download files for offline access when needed
    • ‚úÖ Use Folders page for upload/edit operations (Media is read-only)
    • ‚ùå Don't share sensitive files via public share links

    For Administrators:

    • ‚úÖ Regularly clean up unused profile images to free storage
    • ‚úÖ Organize media library with clear folder structure
    • ‚úÖ Review share link expiration settings for your group
    • ‚úÖ Monitor storage usage and advise users to delete old files
    • ‚ùå Don't delete files without confirming they're truly orphaned
